# 交易条件判断规则

> 本文档是所有交易判断逻辑的唯一来源。**修改代码前，必须先修改本文档。**
>
> 工作流：修改本文档中的规则描述 → 更新代码 → 运行批量验证 → 根据结果继续调优。

---

## 术语表

### 运行与调整

市场走势就是无数个**运行**和**调整**的持续组合。

- **运行** = 释放。价格朝一个方向持续推进，消耗动能。
  - 特征：N字形、连续性、强势动能、斜率越大消耗越快
  - 好比呼气：在消耗积累的能量

- **调整** = 蓄力。价格在一个区间内反复震荡，积累筹码。
  - 特征：W/M形、反复性、弱势动能、斜率与运行方向相反
  - 好比吸气：在积累新的能量

运行和调整交替出现是趋势健康的保障。本系统的核心理念：**不在释放中参与，尽量在蓄力充足后参与**。

### A段

A段是筹码积累充足后的**第一段运行**。识别口诀：找到最后一段调整，锁定最后一段运行，被逆转后确立新的运行方向。这正是我们六维分析要捕捉的机会。

### 关键位

- **平台位**：盘整过程中价格反复触碰的水平价位，好比"中场休息时搭的台子"
- **颈线位**：趋势反转形态（头肩形态）中的关键位，代表"趋势已经改变"

## 总体框架

我们用六个维度来判断一只股票是否具备开仓条件：

| 序号 | 维度 | 简称 | 核心问题 |
|------|------|------|----------|
| 1 | 平台位 | PT | 调整区间内是否形成了被反复验证的关键价位？ |
| 2 | 独立结构 | DL | 是否经历了足够长时间的横盘调整？（由PT反推确定） |
| 3 | 轮廓质量 | LK | 调整的形态是否工整、波浪规则、边界平滑？ |
| 4 | 释放级别 | SF | 调整期间整体是否保持水平，还是已经向某个方向偏移释放了动能？ |
| 5 | 统一区间 | TY | 调整末尾是否出现了K线越来越小的挤压收敛？ |
| 6 | 动能 | DN | 是否出现了有力量的突破K线？ |

**执行顺序**: PT → DL → LK → SF → TY → DN

- 先找PT位（至少2个测试点的水平价位），PT的起止范围确定DL的结构边界
- 找不出PT位 → 不能形成有效DL
- DL ≥90根通过，<90根继续分析后续维度但标注"结构未成熟，待观察"
- 每个维度独立评分为 S/A/B/C 四个等级
- **不做加权打分**，不做综合评级，六个维度的结果直接展示，由人来综合判断

## 第一维度：独立结构 (DL)

### 判断目标

确认股票正在经历一段**水平方向的调整**（盘整），而不是处于单边运行中。调整时间越长，说明筹码交换越充分，后续突破的可信度越高。

### 判断规则

1. DL结构的边界由PT位反推确定（见PT维度）：PT位的起止范围就是DL的结构区间
2. 找不出有效PT位（至少2个测试点的水平价位）→ 不能形成有效DL
3. 判断"走平"的标准：每20根K线为一组，组内价格变化速率不超过每根K线0.3%
4. 允许调整段中间夹杂少量波动（最多容忍15组不平的窗口），只要整体仍然呈现横盘特征
5. **结构边界截断规则**：遇到以下情况时，结构区间应在此处截断，不再向前扩展：
   - 明显的趋势段（连续大幅上涨或下跌，斜率远超盘整阈值）
   - 前一段运行的起点或终点（调整区间不应包含运行段）
   - 价格振幅突然放大到与调整区间不匹配的程度
6. 统计这段调整包含多少根小时K线：
   - **≥ 90根** → 通过（S）
   - **< 90根** → 调整还不到位（C），标注当前根数，继续分析后续维度但标注"待观察"。C不代表否定，可以等多几天达到90根

### 评分

| 等级 | 条件 |
|------|------|
| S | ≥ 90根小时K线，调整充分 |
| C | < 90根，调整不到位，可继续观察等待达到90根 |

DL只看K线数量，不做其他限制。≥90即为S，<90即为C。

### 参考信息（不影响DL评分）

以下信息作为分析备注记录，供人工综合判断时参考，**不影响DL的S/C评分**：

- 调整之前是否有急跌/急涨（变化速率 > 0.5%/K线）
- 调整区间是否有明显倾斜（变化速率 > 0.08%/K线）
- 这些因素会在reasoning中标注，但不改变DL等级

## 第二维度：平台位 (PT)

### 判断目标

在调整区间内找到价格反复验证的**关键价位**（平台位或颈线位）。这些价位被市场反复测试，证明了其作为阻力或支撑的有效性。

### 核心原则

- 平台位的本质是**关键位置、薄弱点**
- 平台位 = "中场休息搭的台子"；颈线位 = "趋势反转的头肩形态"
- 不是一条线，而是一个**区间**

### 判断规则

1. **找候选价位**：对调整区间内的高点和低点进行价格聚类

2. **硬性要求：3次以上支点测试**
   - 支撑位测试：K线最低价接近平台价，但收盘价在平台价之上（从下方弹起）
   - 阻力位测试：K线最高价接近平台价，但收盘价在平台价之下（从上方回落）
   - 不足3次测试 → 不算有效平台位

3. **测试间隔要求**
   - 两次测试之间理想间隔 **20根以上** K线
   - 多次测试最好在整个 DL 期间**比较均匀分布**，而不是集中在某一段
   - 间隔不足会降级

4. **穿越处理**
   - 实体和影线完全不穿越平台位 → 最优
   - 影线稍微超过但实体未穿越 → 可接受
   - 实体穿越（过高点） → 视为瑕疵：
     - 穿越后有≥2次有效测试 → 恢复为A（"虽然过高点，但后续有2次测试"）
     - 穿越后不足2次测试 → 直接降为C（恢复不够，平台无效）

5. **平台区间宽度**：区间范围 = 实际测试点的价格分布范围 + K线实体均宽作为上下缓冲

6. **最后一次测试后的调整要求**：DL中最后一次触碰平台位之后，价格需要先离开平台位走出一段调整结构，然后再出现TY+DN突破。不能最后一次测试完就直接突破。

### 评分规则

| 等级 | 条件 |
|------|------|
| S (优秀) | ≥ 3次测试，实体和影线均不穿越，测试间隔充分且分布均匀 |
| A (一般) | ≥ 3次测试，仅影线穿越；或实体穿越（过高点）后又有≥2次有效测试 |
| B (较差) | 多次影线穿越但仍有测试；或测试间隔不够 |
| C (不满足) | 不足3次测试；或实体穿越后不足2次测试（恢复不够，平台无效） |

### 双平台

- 阻力位（上平台）和支撑位（下平台）分别独立检测、独立评分
- **A股只检测阻力位**（见市场特殊规则），支撑位跳过
- DN 确定方向后：向上突破 → 关注阻力位；向下突破 → 关注支撑位

## 第三维度：轮廓质量 (LK)

### 判断目标

评估调整形态的"颜值"。由于价格需要反复测试平台位，调整区间天然会呈现**波浪形**。好的轮廓应该波浪规则、上下边界平滑、K线大小均匀；差的轮廓则参差不齐、忽大忽小。

### 判断规则

综合评估以下方面（偏向视觉感知，不仅仅是数学指标）：

1. **整体对称性**：调整形态的前半段和后半段是否呈对称/镜像关系
   - 好的调整像呼吸一样有节奏，前后对称
   - 从案例看："虽然中间调整幅度深，但整体走势对称和平衡" → 可以接受

2. **尾部破位检测**：调整区间的尾部是否突破了前期低点（做多方向）
   - 尾部下跌幅度过大、突破了前期低点 → 明显降分
   - 尾部走势太凌厉 → 降分但不如破位严重

3. **中间段密集度**：调整区间中间部分的K线是否紧密
   - 中间太松散、密集度不够 → 降分
   - K线均匀密集 → 加分

4. **波浪规则性**：波浪是否对称、有节奏感
   - 上轨（滚动最高价）和下轨（滚动最低价）的变化是否平缓
   - 各根K线的振幅是否接近

5. **边界平滑度**：整体轮廓的上下边界是否平整
   - 是否有突然冒出的异常大K线破坏整体形态
   - 振幅超过"均值 + 2倍标准差"的K线算异常

### 评分规则

| 等级 | 条件 |
|------|------|
| S (优秀) | 整体对称、波浪规则、K线密集均匀、无尾部破位 |
| A (一般) | 整体尚可，可能某一方面有瑕疵（如尾部稍凌厉但整体对称） |
| B (较差) | 尾部破位（突破前期低点）；或中间太松散密集度不够 |
| C (不满足) | 多项指标不达标，形态杂乱 |

### 评分理由示例（来自实际案例）

- S: 形态工整，前后对称，K线密集均匀
- A: "前面都挺好，尾部走势太凌厉"；"中间调整幅度深，但整体对称和平衡"
- B: "尾部下跌幅度过大，突破了前期低点"；"中间太松散，密集度不够"
- C: 多项指标同时不达标

**窄幅结构惩罚**：如果调整振幅不到均价的3%，评分阈值上调，因为过窄的调整可能是假调整。

## 第四维度：释放级别 (SF)

### 判断目标

评估调整结构的**尾部**是否向突破方向蹭上去了。好的调整应该是水平的，动能完全蓄积而不是提前释放。如果尾部向突破方向蹭了一段，说明动能已经提前消耗了一部分。

### 核心原则

- 运行 = 释放（消耗动能），调整 = 蓄力（积累动能）
- SF评估的是：这段调整是"纯粹的蓄力"，还是"蓄力的同时尾部已经在释放"？
- 因为要反复测试PT（3次以上），LK自然是波浪形，但波浪的尾部不能明显向突破方向漂移

### 判断规则

**方向相关**：SF需要结合方向来判断。做多时看尾部是否向上蹭，做空时看尾部是否向下蹭。

**判断偏向视觉感知**——观察DL结构的尾部，看起来是平的、还是往突破方向蹭上去了？

| 级别 | 特征 | 含义 |
|------|------|------|
| 1st (最佳) | 调整结构整体水平，尾部没有向突破方向蹭 | 动能完全蓄积，条件满足可直接做 |
| 2nd (次优) | 尾部向突破方向蹭了一点，幅度不大 | 动能有一定消耗，需要再等一段调整才能参与 |
| 3rd (需重等) | 尾部向突破方向蹭了很多，幅度很大 | 动能已经消耗完，需要重新等待一个新的独立结构 |

### 非1st评分原因示例

- 2nd: "尾部向上蹭，幅度不大，动能有一定消耗"
- 3rd: "尾部向上蹭幅度很大，动能已消耗完"

## 第五维度：统一区间 (TY)

### 判断目标

检测调整**最末尾**是否出现了K线越来越小、几乎不动的"挤压"状态。这是多空双方力量即将打破平衡的前兆，是最佳的入场时机信号。

### 核心原则

- 释放前的迹象：挤压到极限，筹码集中度优势
- 统一区间和启动K线（DN）之间最多允许1根K线间隙，解决交易润滑问题

### 判断规则

1. 在调整区间的**最后30根K线**里扫描
2. 找到连续的"小K线"序列（K线振幅 < 整体ATR的60%）
3. 允许中间最多夹杂1根非小K线（偶尔的异常不影响整体判断）
4. 检查挤压区的斜率是否水平（不能往某个方向倾斜太多）
5. 挤压区必须**紧贴调整末端**（距离不超过1根K线）

### 评分规则

| 等级 | 条件 |
|------|------|
| S (优秀) | ≥ 4根小K线，低斜率（< 0.01%），紧贴末端 |
| A (一般) | ≥ 4根小K线，斜率可放宽（< 0.02%），紧贴末端 |
| B (较差) | ≥ 3根小K线（需其他维度整体好才有意义） |
| C (不满足) | 小K线不足3根，或斜率过大，或离末端太远 |

### 经验型放宽

- 整体条件都非常好时，统一区间的最小边界可以放宽到3根K线（B级）
- TY非常好且其他条件也不错时，TY与启动K线之间即使走势不完全符合规则，也可经验判断接受

## 第六维度：动能 (DN)

### 判断目标

评估是否出现了一根（或几根合并的）**有力量的突破K线**，体现某一方筹码占据压倒性优势，实质性突破了调整区间和平台位。

### 核心原则

- 好的突破是果断的：最好单根大阳线（或大阴线）一举穿越平台
- 合并K线越多，说明动能越犹豫，质量越差
- 必须是实质性突破，不是刚刚蹭到平台位边缘

### 判断规则

1. **在哪里找**：从调整结构末端开始，往后看最多10根K线
2. **突破方向**：向上和向下都检测
   - 向上：收盘价突破调整上沿，K线实体足够大
   - 向下：收盘价跌破调整下沿，K线实体足够大
3. **合并K线**：单根没突破时，尝试合并2~3根连续K线来看
4. **平台位突破确认**：
   - 基本要求：大阳线实体穿越平台位区间
   - 如果PT有瑕疵（有实体穿越但经2次测试恢复有效），DN的收盘价需要超越该穿越高点
   - 如果平台位前有小高点，最好也突破该小高点
5. **如果没有突破**：标记为"等待中(Pending)"

### 力度衡量

- **力度比** = 突破K线实体 / 挤压区K线平均振幅（相对于TY要有绝对力度）
- **放量比** = 突破K线成交量 / 挤压区平均成交量
- 整体幅度不固定，要从整个图形的视觉感知来判断是否"够强"

### 评分规则

| 等级 | 条件 |
|------|------|
| S (优秀) | 单根突破，力度强劲，实体穿越平台位，明显放量 |
| A (一般) | 单根突破，实体穿越平台位 |
| B (较差) | 2根合并突破，力度可接受 |
| C (不满足) | 3根或更多合并，或力度不足，或未穿越平台位 |

### Pending 状态

- DN还没出现突破K线时，标记为 Pending
- Pending 不代表条件差，只是还没到突破时机
- 其他五个维度的评分仍然有参考价值

## 综合展示

**不做加权打分，不做综合评级。** 六个维度的结果直接展示，由人来综合判断是否操作。

### 通用规则：非S评分必须附带原因

每个维度的评分，**除了S（或1st）以外**，A/B/C（或2nd/3rd）都必须附带文字描述原因。S级和1st级不需要特别说明。

### 结论输出格式

```
{方向}：DL{等级}/PT{等级}/LK{等级}/SF{级别}/TY{等级}/DN{等级}
仓位：{1R/0.5R/等待/不做}（原因）
```

示例：
```
看多：DLS/PTS/LKA(尾部稍凌厉)/1st/TYS/DNS
仓位：1R（所有维度均≥A）

看多：DLS/PTC(触碰不足3次)/LKB(中间松散)/2nd(尾部向上蹭)/TYA/DNS
仓位：0.5R（TY≥A+DN=S，可半仓做）

待定(多)：DLC(72根)/PTB(间隔不足)/LKA(对称性略差)/1st/TYA/DN待定
仓位：等待（DN尚未触发）
```

- DL未达90根时标注当前根数，如 `DLC(72根)`
- DN未出现突破时标注 `DN待定`
- SF用1st/2nd/3rd表示
- **非S/非1st评分后面括号内附带简短原因**

## 仓位管理

仓位档位由除DL之外的五个维度（PT/LK/SF/TY/DN）综合决定。

### R值

- **R = 单笔最大止损金额**（当前固定1000元，未来可能改为总仓位的1%~2%）
- 买入股数 = R / (买入价 - 止损价)
- 这样不管止损幅度多大，每笔最大亏损始终控制在R以内

### 仓位档位

| 档位 | 条件 | 说明 |
|------|------|------|
| **1R** | 所有维度均在A或以上（SF=1st视为≥A） | 标准满仓位，最佳条件 |
| **0.5R** | TY≥A + DN≥S，即使PT/LK/SF不够好（甚至为C） | 半仓做，TY+DN的强势可弥补其他不足 |
| **等待** | DN尚未触发（Pending）；或SF=2nd+其他维度一般 | 继续观察，等待时机 |
| **不做** | SF=3rd；或关键维度不达标且TY/DN也不够好 | 条件不满足，放弃本次机会 |

### 仓位判断优先级

1. DN=Pending → **等待**（最高优先级，突破尚未发生）
2. SF=3rd → **不做**（动能已消耗完，无法挽回）
3. DL<90根 → **等待**（结构未成熟，可继续观察）
4. 所有维度均≥A（SF=1st） → **1R**
5. TY≥A + DN=S → **0.5R**（即使PT/LK/SF为C）
6. SF=2nd → **等待**（需再等一段调整）
7. 其余 → **不做**

## 市场特殊规则

### A股

- **只能做多**，不生成看空结论
- 如果突破方向向下 → 标注"A股不做空"
- **PT只检测阻力位（上平台），不检测支撑位** — 因为A股只做多，只关心向上突破阻力位
- SF方向固定为"做多"，只看尾部是否向上蹭

### 美股

- 可以做双边
- PT检测阻力位和支撑位，双平台独立评分
- 主结论为确定方向，另一侧作为备注参考
