# 品种库状态流转逻辑

## 一、状态定义

品种库采用 **6 状态模型**，每只品种在生命周期中处于以下状态之一：

| 状态 | 中文 | 说明 |
|------|------|------|
| `pending` | 待入库 | 新添加，等待人工确认 |
| `idle` | 在库中 | 已入库，等待扫描筛选 |
| `watching` | 关注中 | 满足基础关注条件，每小时监控 |
| `focused` | 重点关注 | 满足重点条件，每5分钟监控 |
| `holding` | 持仓中 | 六维达标，已触发模拟下单 |
| `removed` | 已移除 | 人工移除或完全无数据 |

## 二、状态流转图

```
                  人工确认              扫描升级             监控升级             六维达标
  pending ──────────► idle ──────────► watching ──────────► focused ──────────► holding
                       ▲                  │  ▲                │  │                 │
                       │                  │  │                │  │                 │
                       │   不满足关注条件   │  │  不满足重点条件  │  │     平仓          │
                       │◄─────────────────┘  │◄───────────────┘  │                 │
                       │                     │                   │                 │
                       │     不满足关注条件    │                   │                 │
                       │◄────────────────────────────────────────┘                 │
                       │                                                           │
                       │                     平仓（止损/止盈/手动）                   │
                       │◄──────────────────────────────────────────────────────────┘
                       │
                       ▼
                    removed ◄── 人工移除（任何在库状态均可）
                       │
                       └──► pending （人工恢复）
```

## 三、升降级条件

### 六维评级体系

| 维度 | 代码 | 评级范围 | 说明 |
|------|------|----------|------|
| 大浪 | DL | S / A / B / C | 大级别趋势方向 |
| 形态 | PT | S / A / B / C | 价格形态质量 |
| 锁仓 | LK | S / A / B / C | 筹码锁定程度 |
| 安全 | SF | 1st / 2nd / 3rd | 安全边际（越小越好） |
| 时运 | TY | S / A / B / C | 时机判断 |
| 驱动 | DN | S / A / B / C | 基本面驱动力 |

评级优先级（按模块区分）：
- **品种库模块**：仅使用系统算法评级，不受人工标注影响。状态机升降级完全由算法结果驱动。
- **标注模块**：人工标注 > 系统分析结果。用于历史快照回溯，不影响品种库状态流转。

### 升级条件

| 转移路径 | 条件 | 说明 |
|----------|------|------|
| idle → watching | DL=S, PT≥B, LK≥B, SF≤2nd | 大浪最优 + 形态锁仓至少 B + 安全边际在 2nd 以内 |
| watching → focused | DL=S, PT≥A, LK≥A, SF=1st, TY≥A | 形态锁仓升至 A，安全边际最优，时运至少 A |
| focused → holding | DL=S, PT≥A, LK≥A, SF=1st, TY≥A, DN≥A | 在重点基础上，驱动也达到 A，触发模拟下单 |

### 降级条件

| 转移路径 | 条件 | 说明 |
|----------|------|------|
| watching → idle | 不满足 DL=S, PT≥B, LK≥B, SF≤2nd | 任一维度低于关注门槛 |
| focused → watching | 不满足重点条件，但仍满足关注条件 | 降级但保留关注 |
| focused → idle | 不满足重点条件，且不满足关注条件 | 直接回库 |
| holding → idle | 止损/止盈/手动平仓 | 平仓后回到在库中 |

### 移除规则

- **人工移除**：任何在库状态（idle/watching/focused）均可由用户手动移除
- **自动移除**：仅限 `daily_scan` 扫描时发现**完全无数据**的品种（无名称 + 无评级 + 无价格同时满足）
- **恢复**：removed → pending → idle，需人工操作

## 四、调度任务

| 任务 | 频率 | 执行时段 | 作用 |
|------|------|----------|------|
| `focus_monitor` | 每 5 分钟 | 仅交易时段 | 检查 focused 品种：六维达标→下单，不满足→降级 |
| `watch_monitor` | 每 1 小时 | 仅交易时段 | 检查 watching 品种：满足重点条件→升级，不满足关注→降级 |
| `daily_scan` | 每周六 08:00 | 不限 | 扫描 idle 品种池：满足关注条件→升级，无数据→移除 |

### 交易时段定义

| 市场 | 时段 | 时区 |
|------|------|------|
| A股 + 港股 | 周一~周五 09:00 - 16:10 | 北京时间 |
| 美股 | 周一~周五 09:30 - 16:00 | 美东时间（自动适配夏令时/冬令时） |

非交易时段，`focus_monitor` 和 `watch_monitor` 自动跳过，不消耗 API 调用。

## 五、监控流程详解

### focus_monitor（每 5 分钟）

```
遍历所有 focused 品种:
  ├─ 六维全部达标 (SAA+1st+A+A) → 模拟下单 → 转入 holding → 通知
  ├─ 不满足重点条件 (SAA+1st+A):
  │   ├─ 仍满足关注条件 (SBB+≤2nd) → 降级到 watching → 通知
  │   └─ 也不满足关注条件           → 降级到 idle → 通知
  └─ 维持 focused（无变化）
```

### watch_monitor（每 1 小时）

```
遍历所有 watching 品种:
  ├─ 满足重点条件 (SAA+1st+A) → 升级到 focused → 通知
  ├─ 不满足关注条件 (SBB+≤2nd) → 降级到 idle
  └─ 维持 watching（无变化）
```

### daily_scan（每周六 08:00）

```
遍历所有 idle 品种:
  ├─ 完全无数据（无名称+无评级+无价格）→ 移除
  ├─ 满足关注条件 (SBB+≤2nd) → 升级到 watching → 通知
  └─ 不满足条件 → 保持 idle（不会自动移除）
```
